# Security Policy

This document outlines the security policy for the **CV. Kirana Tanjung Pelakar** information system.

## ✅ Supported Versions

Our security policy is to support the latest stable version of the application. Security updates will be applied to the most recent release.

| Version | Supported          |
| ------- | ------------------ |
| 1.2.x   | :white_check_mark: |
| < 1.2.0 | :x:                |

## 🛡️ Reporting a Vulnerability

We take all security vulnerabilities seriously. Thank you for helping us improve the security of our system.

### How to Report
If you discover a security vulnerability, please report it to us by **creating a GitHub Issue** in our repository:
[https://github.com/BotHunting/kirana-tanjung/issues/new](https://github.com/BotHunting/kirana-tanjung/issues/new)

When creating an issue, please provide a detailed description of the vulnerability, including:
- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue.
- Any relevant screenshots, code snippets, or logs.

### Our Commitment
- We will acknowledge receipt of your vulnerability report within 3-5 business days.
- We will provide an initial assessment of the vulnerability's severity.
- We will keep you informed of our progress in resolving the issue.

Given the system's architecture (Vercel, Google Apps Script, Google Sheets), we are particularly interested in reports related to unauthorized data access through the API, Cross-Site Scripting (XSS), or authentication bypass.
